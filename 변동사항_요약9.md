# rag_app_v9.py 변동사항 요약

## 핵심 변경: 로깅 + LLM 자동 평가 + Arize Phoenix 스타일 대시보드

v8의 RAG 파이프라인은 그대로 유지하면서 관찰 가능성(Observability) 레이어 전체를 추가.

---

## 최종 스택
| 구성 | v8 (rag_app_v8.py) | v9 (rag_app_v9.py) |
|------|---------------------|---------------------|
| 임베딩 | text-embedding-3-small | text-embedding-3-small (동일) |
| LLM | gpt-4o-mini | gpt-4o-mini (동일) |
| 검색 파이프라인 | 의도분해 + Pre-filter + Rerank | 동일 |
| 멀티문서 추론 | Step1→2→3 | 동일 |
| **로깅** | 없음 | **JSON 파일 자동 저장** |
| **자동 평가** | 없음 | **LLM 기반 정확도·관련성·환각 판정** |
| **대시보드** | 없음 | **탭 분리 + 지표 + 차트 + 테이블 + CSV** |
| **UI 구조** | 단일 화면 | **탭 2개 (챗봇 / 평가 로그)** |

---

## 변경 사항

### 1. LLM 자동 평가 (`evaluate_answer`)
답변 생성 직후 GPT로 3가지 품질 지표를 자동 채점.
```python
def evaluate_answer(question, context_chunks, answer):
    # 입력: 질문 + 참고 문서 + 생성된 답변
    # 출력:
    # 정확도: 1~5  (문서 내용과 사실적으로 일치하는가)
    # 관련성: 1~5  (질문에 제대로 답하고 있는가)
    # 환각여부: 없음|부분적|있음  (문서에 없는 내용이 포함됐는가)
    # 환각근거: (환각된 내용 요약)
```
**이유:** 사람이 직접 검토하지 않아도 각 답변의 품질을 즉시 수치화. hallucination은 LLM이 문서와 답변을 직접 비교해 판정하므로 단순 키워드 검사보다 신뢰도 높음.

---

### 2. 로그 저장·불러오기 (`save_log` / `load_logs`)
모든 Q&A 세션을 `rag_eval_logs.json`에 누적 저장.
```python
# 저장 항목 (각 질문마다)
{
    "timestamp": "2026-04-17 10:23:45",
    "question": "원본 질문",
    "queries": ["원본", "변형1", "변형2"],
    "retrieved_chunks": [
        {"text": "청크 앞 200자...", "source": "파일명.pdf", "rerank_score": 8.5}
    ],
    "answer": "최종 답변 전문",
    "evaluation": {"정확도": 4, "관련성": 5, "환각여부": "없음", "환각근거": "없음"},
    "latency_ms": 3241,
    "mode": "multidoc"
}
```
**이유:** 세션이 끊겨도 이력이 남아 있어 시간에 따른 품질 추이 분석 가능. JSON 형식이라 외부 분석 도구(pandas, Jupyter 등)로 바로 활용 가능.

---

### 3. 챗봇 화면 — 평가 뱃지 인라인 표시
답변 아래에 4개 지표를 카드 형태로 즉시 표시.
```
⏱ 응답 시간  |  📐 정확도  |  🎯 관련성  |  🟢/🟡/🔴 환각
```
환각이 감지된 경우 경고 배너로 어느 부분이 환각인지 표시.

---

### 4. 평가 로그 대시보드 탭 (Arize Phoenix 스타일)
UI를 `st.tabs`로 2개로 분리. "📊 평가 로그" 탭 구성:

#### 상단 요약 지표 (5개 metric 카드)
```
총 질문 수 | 평균 정확도 | 평균 관련성 | 환각 없음 비율 | 평균 응답 시간
```

#### 시각화 차트 (2개)
```
- 환각 여부 분포: 없음/부분적/있음 bar chart
- 정확도·관련성 추이: 질문 순서별 line chart
```

#### 전체 로그 테이블 (최신 순, 드릴다운 가능)
각 행 expander 클릭 시:
- 좌측: 질문 / 사용된 쿼리 / 검색된 청크 / 최종 답변
- 우측: 정확도·관련성·환각 지표 / 응답 시간 / 추론 모드

#### CSV 다운로드
전체 로그를 UTF-8 BOM CSV로 다운로드 (Excel 열기 호환).

---

## 전체 파이프라인 흐름 (v9)
```
질문 입력 (시간 측정 시작)
  → 쿼리 리라이팅 (의도 분해형)
  → 멀티쿼리 검색 (top 20)
  → Embedding Pre-filter (→ top 10)
  → LLM Rerank (→ top_k)
  → [Step1] 청크 요약
  → [Step2] 공통점·차이점·불확실성 분석
  → [Step3] 최종 구조화 답변 (시간 측정 종료)
  → [평가] LLM 자동 평가 (정확도·관련성·환각)
  → [로깅] rag_eval_logs.json 저장
  → [UI] 답변 + 평가 뱃지 표시
```

## LLM 호출 횟수 비교
| 단계 | v8 | v9 |

| 쿼리 리라이팅 | 1회 | 1회 |
| Rerank | 1회 | 1회 |
| Step1 요약 | 1회 | 1회 |
| Step2 분석 | 1회 | 1회 |
| Step3 답변 | 1회 | 1회 |
| **자동 평가** | — | **+1회** |
| **합계** | **5회** | **6회** |

## 성능 개선 기대 포인트
- 답변 품질의 정량적 추적: 매 질문마다 정확도·관련성·환각 점수가 쌓여 품질 변화 모니터링 가능
- 환각 조기 발견: 답변 생성 즉시 경고가 표시되어 사용자가 잘못된 정보를 그대로 신뢰하는 상황 방지
- 데이터 기반 개선: 로그 CSV로 어떤 질문 유형에서 품질이 낮은지 분석해 청킹·프롬프트 개선에 활용 가능
