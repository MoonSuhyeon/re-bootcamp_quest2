# rag_app_v6.py 변동사항 요약

## 핵심 변경: 답변 품질 제어 (근거 인용 + 확신도 + 구조화 포맷)

v5의 검색·청킹 파이프라인은 그대로 유지하면서, LLM이 출력하는 답변 자체의 신뢰성과 구조를 강제.

---

## 최종 스택
| 구성 | v5 (rag_app_v5.py) | v6 (rag_app_v6.py) |
|------|---------------------|---------------------|
| 임베딩 | text-embedding-3-small | text-embedding-3-small (동일) |
| LLM | gpt-4o-mini | gpt-4o-mini (동일) |
| 벡터 DB | FAISS - 코사인 유사도 | FAISS - 코사인 유사도 (동일) |
| 청킹 | 문단/문장+Overlap / Semantic+Overlap | 동일 |
| 쿼리 처리 | 리라이팅 + 멀티쿼리 | 동일 |
| 리랭킹 | LLM 배치 리랭킹 | 동일 |
| **답변 구조** | 자유 형식 | **요약 → 근거(출처 인용) → 결론(확신도)** |
| **근거 인용** | 없음 | **[출처 N] 번호 인용 강제** |
| **확신도** | 없음 | **높음 / 보통 / 낮음 명시** |

---

## 변경 사항

### 1. 컨텍스트에 [출처 N] 번호 부여
LLM이 어떤 청크를 근거로 썼는지 추적 가능하도록 청크에 번호를 매겨 전달.
```python
# 변경 전 (번호 없이 단순 연결)
context = "\n\n".join(context_chunks)

# 변경 후 (번호 부여 → LLM이 인용 가능)
numbered_context = "\n\n".join(
    [f"[출처 {i+1}]\n{chunk}" for i, chunk in enumerate(context_chunks)]
)
```

---

### 2. 시스템 프롬프트 전면 개정 — 3단계 구조 강제
```python
# 변경 전 (v5: 자유 형식 + 기본 제약만 명시)
"반드시 제공된 컨텍스트만 사용해서 답변하세요."

# 변경 후 (v6: 구조·인용·확신도 모두 명시)
"""
[필수 답변 규칙]
1. 반드시 아래 3단계 구조로 답변:
   **📌 요약**: 핵심 답변 1~2문장
   **📖 근거**: 각 주장에 [출처 N] 인용
   **✅ 결론**: 최종 판단 + 확신도: 높음/보통/낮음 — (이유)

2. 확신도 기준:
   - 높음: 직접적인 근거 있음
   - 보통: 관련 내용 있으나 추론 필요
   - 낮음: 근거 매우 부족 또는 간접적
"""
```
**문제:** v5까지는 LLM이 관련 없는 내용을 자신 있게 서술하거나, 출처 없이 요약만 내놓아 검증이 불가능했음.
**해결:**
- 구조 강제 → 요약/근거/결론이 분리되어 각 주장의 출처를 바로 확인 가능
- 출처 인용 강제 → 근거 없는 주장을 LLM이 스스로 쓸 수 없게 됨
- 확신도 강제 → 정보가 부족한 경우 LLM이 "낮음"으로 표시, 사용자가 과신하지 않도록 유도

---

### 3. UI — 출처 expander 번호 일치
답변에 표시된 [출처 N]과 하단 expander의 번호를 동일하게 맞춤.
```python
# 변경 전: "참고한 문서 내용 보기 [1], [2], ..."
st.caption(f"[{i+1}]")

# 변경 후: "[출처 1], [출처 2], ..." — 답변 본문과 동일한 표기
st.caption(f"[출처 {i+1}]{score_badge}")
```

### 4. `st.write` → `st.markdown` 전환
볼드(`**텍스트**`), 이탤릭 등 구조화된 답변의 마크다운이 올바르게 렌더링되도록 변경.

## 답변 출력 예시
```
📌 요약
계약 해지는 사전 30일 서면 통보 시 가능합니다.

📖 근거
계약서 제5조에 따르면 "일방이 계약을 해지하려면 상대방에게 30일 전 서면으로 통보해야 한다"고 명시되어 있습니다. [출처 1]
단, 귀책사유가 있는 경우 즉시 해지가 가능하다는 조항도 존재합니다. [출처 2]

✅ 결론
일반적인 해지는 30일 전 서면 통보, 귀책사유 발생 시 즉시 해지가 가능합니다.
확신도: 높음 — 두 조항 모두 문서에 명시적으로 기재되어 있음
```

## 성능 개선 기대 포인트
- Hallucination 억제: 출처 인용 강제로 LLM이 문서에 없는 내용을 지어낼 여지 감소
- 사용자 신뢰도 향상: 확신도 표시로 정보의 불확실성을 투명하게 전달
- 검증 용이성: [출처 N] 번호로 답변의 각 주장을 원문과 즉시 대조 가능
