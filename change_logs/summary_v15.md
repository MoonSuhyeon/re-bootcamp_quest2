# rag_app_v15.py 변동사항 요약

## 핵심 변경: LLM 평가 고도화 + AI 오류 원인 분석 시스템

v14의 "실험 도구" 위에, 평가 결과를 단순 수치에서 **구조화된 품질 진단**으로 끌어올리고
"왜 틀렸는가"까지 자동으로 설명하는 AI 오류 원인 분석 시스템 추가.

---

## 최종 스택
| 구성 | v14 (rag_app_v14.py) | v15 (rag_app_v15.py) |
|------|----------------------|----------------------|
| 임베딩 | text-embedding-3-small | 동일 |
| LLM | gpt-4o-mini | 동일 |
| 검색 방식 | Dense + BM25 RRF | 동일 |
| 쿼리 라우팅 | 6개 의도 자동 분류 | 동일 |
| Ablation Study | 6개 Config 비교 실험 | 동일 (평가 지표 확장) |
| **평가 출력 필드** | 정확도·관련성·환각여부·환각근거 | **+ 신뢰도·불일치_항목·누락_정보·개선_제안** |
| **환각 분석 필드** | 환각_주장·유형·근거·발생원인·개선제안 | **+ 심각도(1~10)·원문_인용·수정_제안** |
| **종합 품질 리포트** | 없음 | **build_quality_report() 신규** |
| **에이전트 탭 차트** | 5개 | **8개 (신뢰도·등급·종합점수 추이 추가)** |

## 변경 사항

### 1. `evaluate_answer()` — 평가 시스템 고도화

#### 신규 출력 필드

| 필드 | 설명 | 가능한 값 |
|------|------|-----------|
| `신뢰도` | 전반적 답변 신뢰도 | 높음 / 보통 / 낮음 |
| `불일치_항목` | 문서 내용과 답변이 어긋나는 항목 | 텍스트 또는 "없음" |
| `누락_정보` | 문서에는 있지만 답변이 빠뜨린 핵심 정보 | 텍스트 또는 "없음" |
| `개선_제안` | 답변 품질 향상을 위한 구체적 제안 1문장 | 텍스트 |

#### 강화된 프롬프트
```
정확도: <1~5 정수>
관련성: <1~5 정수>
환각여부: <없음|부분적|있음>
환각근거: <설명>
신뢰도: <높음|보통|낮음>
불일치_항목: <항목 또는 없음>
누락_정보: <정보 또는 없음>
개선_제안: <1문장>
```

---

### 2. `analyze_hallucination_cause()` — 환각 분석 정교화

#### 신규 출력 필드

| 필드 | 설명 |
|------|------|
| `심각도` | 1~10 정수. 10이 가장 심각 |
| `원문_인용` | 환각 주장과 충돌하는 문서 원문 직접 인용 (최대 2문장) |
| `수정_제안` | 해당 환각 부분을 어떻게 고쳐야 하는지 구체적 제안 |

#### 전체 출력 필드 (v15)
```
환각_주장      / 환각_유형 (fabrication|distortion|over-generalization)
심각도          / 근거_출처   / 원문_인용
발생_원인      (insufficient_context|ambiguous_chunk|llm_interpolation)
개선_제안      / 수정_제안
```

---

### 3. `build_quality_report()` — 종합 품질 리포트 (신규)

```python
def build_quality_report(evaluation: dict, hall_cause: dict = None) -> dict:
```

`evaluate_answer()` + `analyze_hallucination_cause()` 결과를 결합해 종합 리포트 생성.

#### 계산 로직
```
overall_score = (정확도 + 관련성) / 2 - 환각_페널티
환각_페널티: 없음=0.0 / 부분적=0.5 / 있음=1.5
범위: 0.0 ~ 5.0
```

#### 등급 기준
| overall_score | 등급 |
|:---:|:---:|
| ≥ 4.5 | A |
| ≥ 3.5 | B |
| ≥ 2.5 | C |
| ≥ 1.5 | D |
| < 1.5 | F |

#### 반환 구조
```python
{
    "overall_score":   3.75,
    "grade":           "B",
    "issues":          ["환각 감지: 부분적 (심각도 5/10)", "누락 정보: 계약 기간 조항"],
    "recommendations": ["출처 명시를 강화하세요", "[환각 수정] 해당 수치를 삭제하세요"]
}
```

### 5. 챗봇 탭 UI 변화

#### 지표 뱃지: 6개 → 7개 (신뢰도 추가)
```
⏱ 응답 | 🔤 토큰 | 📐 정확도 | 🎯 관련성 | 환각 | 🧭 신뢰도 | 📊 NDCG@k
```

#### 🩺 품질 리포트 expander (신규)
- 등급 색상 아이콘 (A=🟢 / B=🔵 / C=🟡 / D=🟠 / F=🔴)
- 감지된 문제 목록
- 개선 제안 목록
- 문서-답변 불일치 항목
- 누락 정보

#### 환각 표시 강화
- 심각도 색상 표시 (≥7=🔴 / ≥4=🟡 / <4=🟢)
- 원문 인용 표시 (최대 80자)
- 수정 제안 경고 박스 (`st.warning`)

### 6. 에이전트 분석 탭 확장

#### 요약 지표: 5개 → 6개 (평균 종합점수 추가)
```
평균 정확도 | 평균 관련성 | 환각 없음 비율 | 환각 감지 수 | 평균 NDCG@k | 평균 종합점수
```

#### 차트 추가 (v15 신규)
| 차트 | 내용 |
|------|------|
| 🧭 신뢰도 분포 | 높음/보통/낮음 bar chart |
| 🏅 답변 등급 분포 | A/B/C/D/F bar chart |
| 📈 종합 품질 점수 추이 | overall_score line chart |

#### 환각 분석 섹션 확장
- 환각 유형 분포 (기존 유지)
- 발생 원인 분포 (기존 유지)
- **평균 심각도** metric 카드 추가 (신규)

#### per-log expander 강화
- 🩺 품질 리포트 (issues + recommendations)
- 🔬 환각 상세 분석 (유형·심각도·발생원인·원문인용·수정제안)
- 신뢰도 metric 추가
- 종합점수 / 등급 metric 추가

## LLM 호출 횟수

| 단계 | v15 |
|------|-----|

| Step2 분석 | 1 |

| **합계** | **7~8회 (v14 동일)** |

> 평가 함수의 출력 필드가 늘었지만 LLM 호출은 1회로 동일. 프롬프트 확장만으로 추가 비용 없음.

## 성능 개선 기대 포인트
- `불일치_항목` + `누락_정보` 누적 → 어떤 유형의 질문에서 문서 커버리지가 부족한지 데이터 수집
- `심각도` 누적 → 파이프라인의 환각 위험도 트렌드 모니터링 가능
- `원문_인용` → 환각 원인을 사람이 즉시 검증 가능한 형태로 제공
- `overall_score` / `grade` → 단일 숫자로 답변 품질 비교 가능 (ablation + 추이 분석)
